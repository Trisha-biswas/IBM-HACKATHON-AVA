# createIndividualPDF Function Documentation

## Overview

The `createIndividualPDF` function creates professionally formatted PDF documents in unique request-specific folders. Each PDF includes custom branding with "Agentic Visa Consultant" header and the traveler's name.

## Function Signature

```javascript
async function createIndividualPDF(
  requestId,      // string - Unique identifier for the request
  travelerName,   // string - Name of the traveler
  documentType,   // string - 'cover-letter' or 'itinerary'
  content,        // string - Document content
  baseDir         // string - Base directory (default: 'public/downloads')
)
```

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `requestId` | string | Yes | Unique identifier for the request (e.g., 'req-12345') |
| `travelerName` | string | Yes | Full name of the traveler |
| `documentType` | string | Yes | Type of document: 'cover-letter' or 'itinerary' |
| `content` | string | Yes | The text content to include in the PDF |
| `baseDir` | string | No | Base directory for downloads (default: 'public/downloads') |

## Return Value

Returns a **Promise** that resolves to a **string** containing the URL path to the generated PDF.

**Example return value:**
```
/downloads/req-12345/cover-letter-Jane-Smith-1714689600000.pdf
```

## Features

### 1. Unique Folder Structure
- Creates a separate folder for each `requestId`
- Folder path: `{baseDir}/{requestId}/`
- Automatically creates folders if they don't exist

### 2. Custom Branding
- **Header**: "Agentic Visa Consultant" in blue (#0f62fe)
- **Subheader**: "Prepared for: {travelerName}"
- **Horizontal line**: Blue separator line
- **Footer**: "Generated by Agentic Visa Consultant on {date}"

### 3. Professional Formatting
- **Page Size**: Letter (8.5" x 11")
- **Margins**: 1 inch on all sides
- **Fonts**: Helvetica and Helvetica-Bold
- **Content Formatting**:
  - Headings: Bold, 12pt
  - Body text: Regular, 11pt, justified
  - Proper paragraph spacing

### 4. File Naming Convention
```
{documentType}-{sanitizedName}-{timestamp}.pdf
```

**Example:**
```
cover-letter-Jane-Smith-1714689600000.pdf
itinerary-John-Doe-1714689700000.pdf
```

### 5. Stream-Based Writing
- Uses `fs.createWriteStream` for efficient file writing
- Handles large documents without memory issues
- Promise-based for async/await compatibility

## Usage Examples

### Example 1: Basic Usage

```javascript
const { createIndividualPDF } = require('./services/fileService');

async function generateDocument() {
  const content = `Dear Visa Officer,

I am writing to apply for a tourist visa...

Sincerely,
Jane Smith`;

  const pdfUrl = await createIndividualPDF(
    'req-2026-05-02-001',
    'Jane Smith',
    'cover-letter',
    content
  );

  console.log(`PDF created: ${pdfUrl}`);
  // Output: PDF created: /downloads/req-2026-05-02-001/cover-letter-Jane-Smith-1714689600000.pdf
}
```

### Example 3: In Express Route Handler

```javascript
router.post('/generate-document', async (req, res) => {
  try {
    const { travelerName, documentType, content } = req.body;
    const requestId = `req-${Date.now()}`;

    const pdfUrl = await createIndividualPDF(
      requestId,
      travelerName,
      documentType,
      content
    );

    res.json({
      success: true,
      downloadUrl: pdfUrl,
      requestId: requestId
    });
  } catch (error) {
    res.status(500).json({
      error: 'Failed to generate PDF',
      details: error.message
    });
  }
});
```

### Example 4: Batch Processing

```javascript
async function generateBatchDocuments(travelers) {
  const results = [];

  for (const traveler of travelers) {
    const requestId = `req-${Date.now()}-${traveler.id}`;

    // Generate cover letter
    const coverLetterUrl = await createIndividualPDF(
      requestId,
      traveler.name,
      'cover-letter',
      traveler.coverLetterContent
    );

    // Generate itinerary
    const itineraryUrl = await createIndividualPDF(
      requestId,
      traveler.name,
      'itinerary',
      traveler.itineraryContent
    );

    results.push({
      traveler: traveler.name,
      requestId,
      coverLetter: coverLetterUrl,
      itinerary: itineraryUrl
    });
  }

  return results;
}
```

## Error Handling

The function throws errors in the following cases:

1. **File System Errors**: Cannot create directory or write file
2. **PDF Generation Errors**: Invalid content or formatting issues
3. **Stream Errors**: Write stream failures

**Example error handling:**

```javascript
try {
  const pdfUrl = await createIndividualPDF(
    requestId,
    travelerName,
    documentType,
    content
  );
  console.log('Success:', pdfUrl);
} catch (error) {
  console.error('Error creating PDF:', error.message);
  // Handle error appropriately
}
```

## Performance Considerations

1. **Memory Efficiency**: Uses streaming for large documents
2. **Concurrent Requests**: Safe for parallel execution
3. **File System**: Creates folders only when needed
4. **Caching**: No caching - generates fresh PDFs each time

## Security Considerations

1. **Input Sanitization**: Traveler names are sanitized to prevent path traversal
2. **Directory Creation**: Uses `recursive: true` for safe folder creation
3. **File Permissions**: Inherits system default permissions
4. **Path Validation**: Ensures files are created within base directory

## Troubleshooting

### Issue: "Failed to create directory"
**Solution**: Check write permissions on base directory

### Issue: "Failed to write PDF"
**Solution**: Ensure sufficient disk space and valid content

### Issue: "Invalid characters in filename"
**Solution**: Function automatically sanitizes names, but check input

### Issue: "PDF not accessible via URL"
**Solution**: Verify Express static middleware is configured:
```javascript
app.use('/downloads', express.static('public/downloads'));
```
